執行方式
1.
在get_pic上寫好P(用來做HDR的照片數量)和要選的照片檔名


2.
gsolve(0,0,1/2,500,[1200 1800],[1,2,3,4,5,6])

第一個 0              佔位參數，不會用到，填多少都可以
第二個 0              佔位參數，不會用到，填多少都可以
第三個 1/2            lambda
第四個 500            表示隨機選擇用來train g function的點的數量，選越多g function noise越少，也能生出越清晰的圖，越多會跑越久
第五個 [1200 1800]    表示希望生成的照片尺寸，越大張會跑越久
第六個 [1,2,3,4,5,6]  表示P張照片的曝光時間，這個例子使用了6張不同曝光時間的相片，故有六個元素

經過等待，就會產生g function
然後會在current directory下生成'output.jpg'

所以如果重複跑，會把前一次生成的'output.jpg'覆蓋掉

如果覺得漂亮想保留要改一下檔名，或者移到其他資料夾喔


2019/03/31:
gsolve(0,0,1/2,500,[1200 1800],[1,2,3,4,5,6],1)

最右邊多了一個參數
 1 表示繪製radiance map，也就是還沒經過tone mapping的HDR影像
 0 表示繪製經過tone mapping的LDR影像
